# Lemniscate Marketing Project Management

Welcome to the Lemniscate Marketing project management repository. This repository serves as the central hub for our agency's document templates and client-specific project files.

## Repository Structure

The repository is organized into two main directories:

### `/templates`
This directory contains standard Markdown templates for all stages of the client lifecycle. These templates ensure consistency and professionalism across all our engagements.

**Pre-sale:**
- `proposal-pitch-deck.md` - For pitching services and outlining value.
- `service-agreement-contract.md` - Formal contract and scope of work.
- `case-study-template.md` - Documenting successful past campaigns.
- `audit-report-template.md` - Presenting initial digital marketing audits.

**Onboarding:**
- `client-onboarding-questionnaire.md` - Gathering essential client information.
- `brand-guidelines-request.md` - Collecting brand assets and voice guidelines.
- `content-approval-workflow.md` - Defining the review and approval process.
- `access-credentials-checklist.md` - Tracking platform access and permissions.

**Monthly Operations:**
- `monthly-report-template.md` - Monthly performance reporting.
- `content-calendar-template.md` - Planning social and content schedules.
- `ad-creative-brief-template.md` - Briefing ad creatives for paid media.
- `meeting-agenda-minutes-template.md` - Structuring meetings and tracking action items.

**Quarterly:**
- `quarterly-business-review-template.md` - QBR presentation structure.
- `strategy-refresh-document.md` - Documenting strategic pivots.

**Project-specific:**
- `website-redesign-brief.md` - Requirements for web design projects.
- `seo-strategy-document.md` - Comprehensive SEO planning.
- `paid-media-plan.md` - Strategy for Google and Meta Ads.
- `social-media-strategy-document.md` - Social media approach and pillars.

**Off-boarding:**
- `handover-document-template.md` - Transferring assets and access back to the client.
- `final-report-results-summary.md` - Summarizing the engagement's overall results.

### `/clients`
This directory holds specific folders for each active or past client. 

- **`/clients/lueur`** - Contains documentation for Lueur Skin Clinic (Melbourne), including their initial proposal and pitch deck.

## Usage

When starting a new project or onboarding a new client:
1. Create a new folder under `/clients/[client-name]`.
2. Copy the relevant templates from `/templates` into the new client folder.
3. Fill out the templates with client-specific information.

*Note: All documents are maintained in Markdown format for easy version control, readability, and consistency.*
