## Context

At the 30-day mark, do a quick health check on the relationship and the work.
Early problems caught here prevent bigger problems at 90 days.

**Client:** PLACEHOLDER_CLIENT_NAME
**Due date:** 30 days after kickoff

This is a brief internal review + a short client touchpoint — not a full
report. The goal is to surface any friction before it becomes a problem.

## Checklist

**Internal Review (before client call)**
- [ ] Are all tasks on track? Flag anything overdue or at risk
- [ ] Has all agreed access been provided?
- [ ] Is the client responsive? (note if communication is slow)
- [ ] Is the scope holding? Note any out-of-scope requests made so far
- [ ] Are we on track for delivering first deliverables on time?

**Client Touchpoint**
- [ ] Book a brief 15-minute check-in call (or send a written update if they prefer)
- [ ] Ask: "Is there anything we could be doing differently to make this easier for you?"
- [ ] Confirm upcoming deliverables and dates
- [ ] Address any concerns raised

**After**
- [ ] Document any issues or feedback in the project file
- [ ] Adjust timelines or task assignments if needed
- [ ] Note any scope creep for the next invoice conversation

## Acceptance Criteria

- Internal health check complete before client touchpoint
- Client touchpoint completed (call or written)
- Any open issues documented and assigned
- Relationship health: Green / Amber / Red status recorded in project file
