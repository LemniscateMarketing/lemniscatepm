## Context

With the keyword map complete, audit each priority page on the site for
on-page SEO hygiene. This is the most directly actionable part of the audit —
most fixes can be implemented without developer involvement.

**Client:** PLACEHOLDER_CLIENT
**Website:** PLACEHOLDER_DOMAIN

Focus on the top 15–20 pages by traffic + commercial intent. Do not attempt to
audit every page — prioritise where the opportunity is.

## Checklist

- [ ] Pull list of top pages from Google Analytics (organic traffic, last 90 days)
- [ ] For each priority page, audit:
  - [ ] Title tag: length (50–60 chars), primary keyword present, compelling
  - [ ] Meta description: length (150–160 chars), includes CTA, not truncated
  - [ ] H1: one per page, matches primary keyword intent
  - [ ] H2/H3 structure: logical hierarchy, secondary keywords used naturally
  - [ ] URL slug: short, descriptive, keyword-rich, no parameters
  - [ ] Internal links: pages link to each other logically; no orphan pages
  - [ ] Image alt text: all images have descriptive alt text
  - [ ] Content length: matches or exceeds competing pages for the same keyword
  - [ ] Schema markup: appropriate type used (Article, Product, FAQ, etc.)
- [ ] Identify pages with cannibalized keywords (2+ pages targeting same term)
- [ ] List top 10 priority fixes sorted by estimated traffic impact

## Acceptance Criteria

- All top 15 pages audited and scored
- Issues logged with page URL, issue type, severity (High / Medium / Low), and fix recommendation
- Keyword cannibalization report complete
- Priority fix list ready for client handoff (non-technical language)
